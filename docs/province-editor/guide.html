프로빈스 에디터 사용 설명서

기후·지형 파이프라인에 넣을 원인값을 작성하고, 최종 제출용 Export ZIP을 만드는 도구입니다.

에디터 열기 Markdown 설명서

1. 에디터의 역할

프로빈스 에디터는 최종 지형을 직접 그림처럼 칠하는 도구가 아니라, 파이프라인이 지형을 계산할 때 참고하는 원인값을 작성하는 도구입니다.

  • 산맥 장벽 강도: 바람과 수분 흐름을 막는 산맥 원인
  • 고도 힌트: 고원·저지대 같은 대략적인 고도 원인
  • 강 시작점: 수문 계산에서 강이 시작될 후보
  • 최종 라벨 강제: 계산 결과와 별개로 특정 프로빈스의 최종 지형/바이옴을 강제

3. 프로빈스 모드와 주 모드

프로빈스 모드

클릭한 프로빈스 하나에 값을 입력합니다. 특정 산맥, 강 시작점, 호수·습지 후보, 프로빈스 예외를 지정할 때 씁니다.

주 모드

클릭한 프로빈스가 속한 주 전체의 하위 프로빈스에 값을 일괄 적용합니다.

현재 파이프라인 v0.1은 state_constraints.yaml을 읽지 않습니다. 그래서 주 모드 결과도 최종적으로는 province_constraints.yaml에 기록됩니다. 즉 주 모드는 “주 값을 따로 저장하는 기능”이 아니라 “프로빈스 대량 편집 기능”입니다.

4. 시뮬레이션 입력

고도 힌트

프로빈스의 대략적인 고도 성격입니다.

미설정 : 값을 기록하지 않음
없음   : 고도 보정 없음
저지대 : 낮은 평야
구릉지 : 완만한 고지·구릉
고지대 : 고원·높은 내륙
산악   : 산악 지대

synthetic_elevation_m 계산에 사용되며, 고도가 높을수록 기온이 내려가고 고원/산지/고산 지형 후보가 됩니다.

산맥 장벽 강도

0.0~1.0 범위의 값입니다. 해당 프로빈스가 습한 바람을 막는 산맥 장벽으로 얼마나 강한지를 뜻합니다.

산맥 한쪽은 습하고 반대쪽은 건조하게 만들고 싶거나, 비그늘/사막권을 유도하고 싶을 때 조절합니다.

산맥 장벽 값 사용을 체크해야 실제로 저장됩니다. 체크하지 않으면 슬라이더 값은 저장되지 않습니다.

수분 보정 / 기온 보정 / 강수 보정

  • 수분 보정: 초기 수분에 더해지는 보정값
  • 기온 보정: 섭씨 기준 기온 보정값
  • 강수 보정: 강수 계산에 더해지는 보정값

셋 다 최종 지형을 직접 고정하는 값이 아니라, 파이프라인 계산을 유도하는 입력값입니다.

판타지 권역

enchanted_forest, corrupted_land, demon_realm 같은 특수 권역 이름을 기록합니다. 최종 바이옴 라벨에 영향을 줄 수 있지만, 기본 terrain 자체를 판타지 terrain으로 만들지는 않습니다.

5. 존재 앵커

  • 강 시작점: 큰 강이 시작될 후보 지점
  • 주요 하천: 시각적으로 중요하거나 큰 하천으로 보고 싶은 구간 표시
  • 호수·종착점: 물이 빠져나가지 않고 모이는 지점 후보
  • 습지 발생점: 습지가 생길 후보 지점
강 시작점과 주요 하천을 반드시 같은 프로빈스에 동시에 켤 필요는 없습니다. 다만 주요 하천만 켜면 의미가 약하고, 강 시작점이 있을 때만 유효합니다.

6. 조건 적용과 조건 삭제

조건 적용은 현재 선택한 프로빈스 또는 주에 시뮬레이션 입력값을 저장합니다.

조건 삭제는 현재 선택한 대상의 시뮬레이션 입력값을 삭제합니다. 주 모드에서는 해당 주 하위 프로빈스들에 적용된 조건을 일괄 삭제합니다.

7. 프로빈스 예외

프로빈스 예외는 일반 계산 결과를 덮어쓰는 강한 기능이며, 프로빈스 모드 전용입니다.

최종 라벨 강제

물리 시뮬레이션에는 계속 참여하지만 마지막 결과 라벨만 강제로 바꿉니다.

최종 라벨 강제가 꺼져 있으면 파이프라인은 지정한 지형·바이옴 값을 강제 결과로 쓰지 않습니다.

허용 지형

plains
forest
hills
mountain
jungle
wetland
desert
tundra
savanna
snow

고급 물리 강제

기온·수분·강수 값을 강제로 고정하는 테스트/예외용 기능입니다.

기후값 강제가 꺼져 있으면 지정한 기온·수분·강수 값은 실제 계산에 반영되지 않습니다.

시뮬레이션 제외

극단 예외용 기능입니다. 일반적인 지형 설정에는 쓰지 않는 것이 좋고, 기후값 강제와 동시에 사용할 수 없습니다.

8. 검증 메시지

에디터는 Export 전에 경고/오류를 검사합니다. 오류가 있으면 Export가 막힙니다.

기후값 강제와 시뮬레이션 제외는 동시에 사용할 수 없습니다.
최종 라벨 강제가 꺼져 있으면 파이프라인은 그 값을 강제 결과로 쓰지 않습니다.
기후값 강제가 꺼져 있으면 지정한 기온·수분·강수 값은 실제 계산에 반영되지 않습니다.
주요 하천은 강 시작점이 있을 때만 유효합니다.
호수·종착점과 강 시작점이 같이 지정되어 있습니다. 이 경우 호수·종착점이 우선됩니다.

9. 저장과 Export

  • 자동 저장: 브라우저 localStorage에 임시 저장
  • 작업 저장 JSON: 중간 저장/백업용 파일
  • 작업 불러오기: 작업 저장 JSON 복원
  • 자동 저장본 삭제: 브라우저 자동 저장본 삭제
  • Export ZIP: 최종 제출용 파일

Export ZIP 구성:

export_manifest.json
province_constraints.yaml
province_overrides.yaml
state_constraints.yaml

현재 파이프라인에서 중요한 파일은 province_constraints.yaml과 province_overrides.yaml입니다. state_constraints.yaml은 현재 v0.1 파이프라인에서 읽지 않으며, 호환용 빈 파일에 가깝습니다.

10. 담당자 작업 순서

  1. 에디터 접속
  2. 편집자 ID 입력
  3. 프로빈스 모드 또는 주 모드 선택
  4. 담당 지역 클릭
  5. 고도 힌트, 산맥 장벽, 강 시작점 등 원인값 입력
  6. 조건 적용
  7. 필요하면 프로빈스 예외 설정
  8. 검증 메시지 확인
  9. 작업 저장 JSON으로 중간 백업
  10. 작업 완료 후 Export ZIP 생성
  11. Export ZIP을 총괄 담당자에게 전달
