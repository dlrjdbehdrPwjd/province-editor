# 프로빈스 에디터 사용 설명서

## 1. 에디터의 역할

프로빈스 에디터는 빅토리아3 판타지 맵의 기후·지형 파이프라인에 넣을 **원인값**을 작성하는 도구다.

이 도구에서 칠하는 값은 최종 지형 결과가 아니라, 파이프라인이 지형을 계산할 때 참고하는 입력값이다.

예를 들어:

- 산맥 장벽 강도: 바람과 수분 흐름을 막는 산맥 원인
- 고도 힌트: 고원·저지대 같은 대략적인 고도 원인
- 강 시작점: 수문 계산에서 강이 시작될 후보
- 최종 라벨 강제: 계산 결과와 별개로 특정 프로빈스의 최종 지형/바이옴을 강제

## 2. 접속 주소

GitHub Pages 배포 주소:

```text
https://dlrjdbehdrpwjd.github.io/province-editor/
```

## 3. 기본 조작

왼쪽 도구막대:

```text
프  : 프로빈스 선택 모드
주  : 주 선택 모드
+   : 확대
-   : 축소
1   : 1배
2   : 2배
4   : 4배
8   : 8배
⛶   : 전체 보기
×   : 선택 해제
```

단축키:

```text
P       : 프로빈스 모드
S       : 주 모드
= 또는 + : 확대
-       : 축소
1       : 1배
2       : 2배
3       : 4배
4       : 8배
5       : 전체 보기
X       : 선택 해제
```

## 4. 프로빈스 모드와 주 모드

### 프로빈스 모드

클릭한 프로빈스 하나에 값을 입력한다.

사용 예:

- 특정 산맥 줄기 설정
- 강 시작점 지정
- 특정 호수·습지 후보 지정
- 특정 프로빈스만 최종 지형 강제

### 주 모드

클릭한 프로빈스가 속한 주 전체의 하위 프로빈스에 값을 일괄 적용한다.

주의:

- 현재 파이프라인 v0.1은 `state_constraints.yaml`을 읽지 않는다.
- 그래서 주 모드 결과도 최종적으로는 `province_constraints.yaml`에 기록된다.
- 즉 주 모드는 “주 값을 따로 저장하는 기능”이 아니라 “프로빈스 대량 편집 기능”이다.

사용 예:

- 큰 사막권 전체에 건조 보정
- 넓은 고원 지대에 고도 힌트 적용
- 국가 담당자가 자기 담당 주를 빠르게 입력

## 5. 시뮬레이션 입력

### 고도 힌트

프로빈스의 대략적인 고도 성격이다.

값:

```text
미설정 : 값을 기록하지 않음
없음   : 고도 보정 없음
저지대 : 낮은 평야
구릉지 : 완만한 고지·구릉
고지대 : 고원·높은 내륙
산악   : 산악 지대
```

파이프라인 영향:

- `synthetic_elevation_m` 계산에 사용된다.
- 고도가 높을수록 기온이 내려간다.
- 고원/산지/고산 지형 후보가 된다.
- 산맥 장벽 강도와는 다른 값이다.

핵심:

```text
고도 힌트 = 이 땅이 얼마나 높은 곳인가
산맥 장벽 강도 = 이 땅이 바람과 수분 흐름을 얼마나 막는가
```

### 산맥 장벽 강도

0.0~1.0 범위의 값이다.

해당 프로빈스가 습한 바람을 막는 **산맥 장벽**으로 얼마나 강한지를 뜻한다.

파이프라인 영향:

- 산맥 장벽 계산에 사용된다.
- 바람이 산맥을 만나면 풍상측 강수가 증가한다.
- 산맥 너머는 수분이 줄어 비그늘이 생길 수 있다.
- 산악 지형 후보를 강화한다.

유저가 이 값을 조절해야 하는 경우:

- 바람을 막는 큰 산맥을 만들고 싶을 때
- 산맥 한쪽은 습하고 반대쪽은 건조하게 만들고 싶을 때
- 대륙 내부에 비그늘/사막권을 만들고 싶을 때
- 산악 지형 판정을 강하게 유도하고 싶을 때

`산맥 장벽 값 사용`을 체크해야 실제로 저장된다.

체크하지 않으면 슬라이더 값은 저장되지 않는다.

### 수분 보정

해당 프로빈스의 초기 수분에 더해지는 보정값이다.

사용 예:

- 습한 숲
- 마력 습지
- 강 주변의 습한 지역
- 비정상적으로 건조한 지역

주의:

- 최종 강수량을 직접 확정하는 값이 아니다.
- 수분 전파 계산의 입력값이다.

### 기온 보정

섭씨 기준 기온 보정값이다.

사용 예:

- 화산 지대, 온천 지대: 양수
- 극지 마법권, 저주받은 냉기 지대: 음수

주의:

- 일반적인 위도/계절/고도 계산 위에 더해지는 보정이다.
- 최종 지형을 직접 고정하지 않는다.

### 강수 보정

해당 프로빈스 강수 계산에 더해지는 보정값이다.

사용 예:

- 항상 비가 많은 지역
- 건조 마법이 걸린 지역
- 세계수 주변처럼 강수량이 특수한 권역

주의:

- 최종 지형을 직접 고정하지 않는다.
- 결과는 이후 정규화와 바이옴 판정을 거친다.

### 판타지 권역

특수 권역 이름을 기록한다.

예:

```text
enchanted_forest
corrupted_land
demon_realm
world_tree
```

파이프라인 영향:

- 최종 바이옴 라벨을 판타지 권역에 맞게 바꿀 수 있다.
- 기본 지형 terrain 자체를 임의의 판타지 terrain으로 만들지는 않는다.

## 6. 존재 앵커

존재 앵커는 “이 지형 요소가 여기 있다”는 힌트다.

### 강 시작점

큰 강이 시작될 후보 지점이다.

사용 예:

- 산맥 풍상측
- 고원 가장자리
- 세계수 주변

질문이 많았던 부분:

```text
강 시작점과 주요 하천을 반드시 같은 프로빈스에 동시에 켤 필요는 없다.
```

강 시작점은 “여기서 강이 시작될 수 있다”는 원인값이다.

### 주요 하천

시각적으로 중요하거나 큰 하천으로 보고 싶은 구간 표시다.

주의:

- 주요 하천만 켜면 의미가 약하다.
- 강 시작점이 있을 때만 유효하다.
- 현재 MVP에서는 강 시작점과 함께 쓰는 보조 표시 성격이 강하다.

### 호수·종착점

물이 빠져나가지 않고 모이는 지점 후보다.

사용 예:

- 내륙 호수
- 염호
- 폐쇄 분지

주의:

- 호수·종착점과 강 시작점을 동시에 켜면 호수·종착점이 우선된다.

### 습지 발생점

습지가 생길 후보 지점이다.

사용 예:

- 삼각주
- 강 하류
- 저지대 늪
- 마법 습지

## 7. 조건 적용과 조건 삭제

### 조건 적용

현재 선택한 프로빈스 또는 주에 시뮬레이션 입력값을 저장한다.

프로빈스 모드:

```text
선택한 프로빈스 하나에 저장
```

주 모드:

```text
해당 주의 하위 프로빈스 전체에 저장
```

### 조건 삭제

현재 선택한 대상의 시뮬레이션 입력값을 삭제한다.

주 모드에서 삭제하면 해당 주 하위 프로빈스들에 적용된 조건을 일괄 삭제한다.

## 8. 프로빈스 예외

프로빈스 예외는 일반 계산 결과를 덮어쓰는 강한 기능이다.

이 영역은 **프로빈스 모드 전용**이다.

### 최종 라벨 강제

최종 지형/바이옴 라벨을 강제한다.

파이프라인 영향:

- 물리 시뮬레이션에는 계속 참여한다.
- 마지막 결과 라벨만 강제로 바꾼다.

예:

```text
이 프로빈스는 계산상 숲이 나와도 최종 지형은 산악으로 고정
```

주의:

```text
최종 라벨 강제가 꺼져 있으면 파이프라인은 지정한 지형·바이옴 값을 강제 결과로 쓰지 않는다.
```

### 지형

강제할 최종 terrain이다.

현재 허용 지형:

```text
plains
forest
hills
mountain
jungle
wetland
desert
tundra
savanna
snow
```

출력하지 않는 지형:

```text
ocean
lakes
river
farmland
urban
docks
impassable
판타지 전용 terrain
```

### 바이옴

강제할 최종 biome 이름이다.

바이옴은 지형과 독립 채널이다.

즉:

```text
force_biome을 넣는다고 terrain이 자동으로 다시 계산되지는 않는다.
force_terrain을 넣는다고 biome이 자동으로 바뀌지는 않는다.
```

### 고급 물리 강제

기온·수분·강수 값을 강제로 고정하는 기능이다.

일반 사용자는 되도록 쓰지 않는 것이 좋다.

사용 예:

- 테스트용
- 특정 마법 권역의 기후값을 직접 고정
- 파이프라인 검증용

주의:

```text
기후값 강제가 꺼져 있으면 지정한 기온·수분·강수 값은 실제 계산에 반영되지 않는다.
```

### 시뮬레이션 제외

극단 예외용 기능이다.

해당 프로빈스를 일반 기후 계산에서 제외한다.

주의:

- 일반적인 지형 설정에는 쓰지 않는 것이 좋다.
- 기후값 강제와 동시에 사용할 수 없다.

## 9. 검증 메시지

에디터는 Export 전에 경고/오류를 검사한다.

오류가 있으면 Export가 막힌다.

대표 메시지:

```text
기후값 강제와 시뮬레이션 제외는 동시에 사용할 수 없습니다.
```

```text
최종 라벨 강제가 꺼져 있으면 파이프라인은 그 값을 강제 결과로 쓰지 않습니다.
```

```text
기후값 강제가 꺼져 있으면 지정한 기온·수분·강수 값은 실제 계산에 반영되지 않습니다.
```

```text
주요 하천은 강 시작점이 있을 때만 유효합니다.
```

```text
호수·종착점과 강 시작점이 같이 지정되어 있습니다. 이 경우 호수·종착점이 우선됩니다.
```

## 10. 저장과 Export

### 자동 저장

브라우저 `localStorage`에 자동으로 임시 저장된다.

특징:

- 같은 브라우저에서만 복원된다.
- 다른 사람에게 전달하는 용도가 아니다.
- 브라우저 캐시 삭제 시 사라질 수 있다.

### 작업 저장 JSON

현재 작업 상태를 JSON 파일로 저장한다.

용도:

- 중간 저장
- 다음에 이어서 작업
- 실수 대비 백업

주의:

```text
작업 저장 JSON은 파이프라인 입력물이 아니다.
```

### 작업 불러오기

작업 저장 JSON을 다시 불러온다.

### 자동 저장본 삭제

브라우저에 남은 자동 저장본을 삭제한다.

### Export ZIP

최종 제출용 파일이다.

Export ZIP 안에는 다음이 들어간다.

```text
export_manifest.json
province_constraints.yaml
province_overrides.yaml
state_constraints.yaml
```

현재 파이프라인에서 중요한 파일:

```text
province_constraints.yaml
province_overrides.yaml
```

`state_constraints.yaml`은 현재 v0.1 파이프라인에서 읽지 않으며, 호환용 빈 파일에 가깝다.

## 11. 담당자가 작업하는 기본 순서

1. 에디터 접속
2. 편집자 ID 입력
3. 프로빈스 모드 또는 주 모드 선택
4. 담당 지역 클릭
5. 고도 힌트, 산맥 장벽, 강 시작점 등 원인값 입력
6. 조건 적용
7. 필요하면 프로빈스 예외 설정
8. 검증 메시지 확인
9. 작업 저장 JSON으로 중간 백업
10. 작업 완료 후 Export ZIP 생성
11. Export ZIP을 총괄 담당자에게 전달

## 12. 추천 사용 방식

일반 담당자는 아래 위주로 쓰는 것을 추천한다.

```text
고도 힌트
산맥 장벽 강도
수분/기온/강수 보정
강 시작점
호수·종착점
습지 발생점
판타지 권역
```

주의해서 써야 하는 것:

```text
최종 라벨 강제
고급 물리 강제
시뮬레이션 제외
```

강제 기능은 “계산을 유도하는 값”이 아니라 “계산 결과를 덮어쓰는 값”이므로 꼭 필요한 경우에만 사용한다.
